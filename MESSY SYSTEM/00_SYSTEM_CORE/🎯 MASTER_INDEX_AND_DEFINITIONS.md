## 🔷 SYSTEM OVERVIEW

### What is MESSY SYSTEM?

The MESSY SYSTEM folder contains **678 files** representing the complete curriculum development system. It includes:

- **SSOT (Single Source of Truth) Documents** - Master definitions
- **Pipeline Indicators** - Files showing progression through stages (using `>>` notation)
- **Templates** - Reusable structures for creating content
- **Research & Content** - Source materials and developed lessons
- **Images & Assets** - Visual materials organized by element/lesson

### Core Philosophy

> **The Geometric Elements are the Unmoved Mover**
> Every lesson, document, and artifact traces back to ONE geometric element (circle, triangle, square, spiral, radial, polygon, grid, fractal).

**One Per:** Each geometric element

### 2. **Ritual Snapshot Document**

**Template:** Billboard_Ritual_Snapshot.md
**Purpose:** Deep dive into ONE key ritual showing embodied cognition
**Contains:**
- Complete ritual sequence with timing
- Material objects used
- Body actions (kinesthetic, tactile, perceptual)
- Frequency × Radius analysis
- Cognitive outcomes

**One Per:** Major ritual (1-2 per element)

## 🚀 HOW TO USE THIS SYSTEM

### For Creating New Content:

1. **Start with Element_Template.md**
   - Define the geometric element fully
   - Location: `01_TEMPLATES/Element_Template.md`

2. **Create Ritual Snapshot using Billboard_Ritual_Snapshot.md**
   - Choose 1-2 key rituals
   - Show embodied cognition clearly
   - Location: `01_TEMPLATES/Billboard_Ritual_Snapshot.md`

3. **Build Day A Lesson using DayA_Lesson_Table.md**
   - Material culture focus
   - Include ritual analysis
   - Location: `01_TEMPLATES/DayA_Lesson_Table.md`

4. **Build Day B Lesson using DayB_Lesson_Table.md**
   - Math + innovation focus
   - Bridge to Day A embodiment
   - Location: `01_TEMPLATES/DayB_Lesson_Table.md`

5. **Compile into eTextbook_Chapter.md**
   - This becomes MASTER SSOT
   - All above sections combined
   - Location: `02_ELEMENTS_REFERENCE/[ELEMENT]/eTextbook-[ELEMENT].md`

3. **Search for >> indicators** to find pipeline stage
4. **Check 02_ELEMENTS_REFERENCE/** for completed elements
